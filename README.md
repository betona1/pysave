# ShotKey — 전역 핫키 자동 화면 캡처 + 영역 동영상 녹화

핫키를 누르면 **대상 창(또는 좌표)에 페이지 넘김 키(기본 →)** 를 보낸 뒤,
지정한 영역을 **자동으로 스크린샷 저장**합니다. 전자책·만화·PDF 연속 캡처 자동화용.
추가로 **지정한 영역을 시스템 소리와 함께 동영상(.mp4)으로 녹화**할 수 있습니다.

## 바로 실행 (설치 없이)

파이썬이 없어도 됩니다. [Releases](https://github.com/betona1/pysave/releases) 에서
`ShotKey.exe` 를 내려받아 더블클릭하면 바로 실행됩니다(Windows).

> exe 안에 파이썬·ffmpeg 까지 모두 들어 있어 용량이 큽니다. 첫 실행은 압축 해제로
> 몇 초 걸릴 수 있습니다. `captures/` 등 저장 폴더는 **처음 사용할 때 자동으로 생성**됩니다.

## 소스로 실행 (개발자)

```bash
pip install -r requirements.txt
python main.py
```

## 사용 순서 (UI)

1. **모니터** — `모니터 식별` 을 누르면 각 화면에 번호가 뜹니다. 대상 모니터를 고르고,
   필요하면 `선택 모니터 전체 캡처 영역으로` 로 그 모니터 전체를 캡처 영역으로 설정합니다.
2. **캡처 영역** — `영역 드래그 지정` 으로 원하는 사각형만 선택할 수도 있습니다(멀티모니터 가상좌표 지원).
3. **키를 보낼 대상**
   - `열린 창 선택`: 현재 떠 있는 프로그램 목록에서 골라, 그 창을 포커스한 뒤 키를 보냅니다.
   - `좌표 클릭`: 저장한 좌표를 클릭해 포커스합니다.
4. **동작 설정** — 페이지 넘김 키(기본 `right`), **반복 횟수**, **딜레이(초)** 설정.
5. **핫키 / 저장 경로** — `핫키 입력` 으로 트리거 키를 잡고(예: F8), 저장 폴더 지정.
6. `● 핫키 시작` → 핫키를 누를 때마다 캡처 시퀀스 실행. `지금 1회 실행(테스트)` 으로 미리 확인.
7. `PNG → PDF 생성` — `captures/page_001.png …` 를 순서대로 묶어 PDF 생성.

## 결과물

- `captures/page_001.png`, `page_002.png` … (0패딩 순번, 이어찍기 시 마지막 번호+1부터)
- `captures/shotkey_HHMMSS.pdf` (PDF 병합 시)
- `<저장폴더>/<폴더명>_1.mp4` (동영상 녹화 시, 소리 포함)

## 동영상 녹화 (설정한 캡처 영역)

캡처 영역(2번에서 지정한 사각형)을 그대로 **동영상 .mp4 로 녹화**합니다.

1. **동영상 저장 폴더** — 비워두면 사진 저장 폴더에 함께 저장됩니다.
2. **파일명** — 기본은 `폴더명_1.mp4`, 같은 이름이 있으면 자동으로 번호가 올라갑니다(`자동 이름` 버튼).
3. **프레임(fps)** — 1~60. 화면 녹화는 보통 15fps 로 충분합니다.
4. **시스템 소리 함께 녹음** — 체크하면 **스피커로 나가는 소리(WASAPI 루프백)** 를 같이 담습니다.
   - 루프백 장치가 없거나 완전 무음이면 **소리 없이 영상만** 저장하고 안내를 남깁니다.
5. **자동 종료** — 시/분/초로 지정하면 그 시간이 지나면 녹화가 자동으로 멈춥니다(모두 0 = 무제한).
6. `⏺ 영역 동영상 녹화 시작` → 다시 누르면(`⏹ 녹화 정지`) 멈추고 파일을 마무리합니다.

> 영상 길이는 **실제 경과 시간 기준으로 fps 를 보정**해 저장하므로 소리와 어긋나지 않습니다.
> 마무리 단계에서 번들 ffmpeg 로 영상+소리를 합쳐 최종 mp4 를 만듭니다.

## exe 빌드 (Windows, 개발자용)

```bash
pyinstaller --noconsole --onefile --name ShotKey ^
  --exclude-module PyQt5 ^
  --collect-all soundcard ^
  --collect-all imageio_ffmpeg ^
  --collect-submodules pynput ^
  --collect-submodules pyautogui ^
  main.py
```

`soundcard`(소리 녹음)와 `imageio_ffmpeg`(번들 ffmpeg)는 지연 import 라서
`--collect-all` 로 명시해야 exe 에 포함됩니다. PyQt5 가 함께 설치돼 있으면
`--exclude-module PyQt5` 로 Qt 바인딩 충돌을 막습니다.

## 끝 페이지 → 반복 횟수 / 자동 PDF

- **끝 페이지** 와 **한 화면당 쪽수**(2쪽씩 보기면 2) 를 넣으면 반복 횟수가 자동 계산됩니다
  (`끝페이지 // 쪽수 + 1`, 예: 566쪽·2쪽보기 → 284회).
- **완료 후 자동 PDF 생성** 체크 시, 캡처가 끝나면 저장 폴더 이름으로 PDF 가 자동 생성됩니다.
- **딜레이** 는 0~600초까지(느린 뷰어는 3초 이상 권장). 모든 설정은 종료 시 자동 저장되어 재실행해도 유지됩니다.

## 대상 창 선택 (중요)

- 밀리의서재 같은 뷰어 앱은 **창 제목이 현재 책 제목으로** 표시됩니다.
  따라서 대상 창은 앱 이름("밀리의서재")이 아니라 **펼친 책 제목(예: "나의 사주명리")** 을 골라야 합니다.
- 창 활성화는 **창 크기를 바꾸지 않습니다**(최소화된 창만 복원). 최대화 상태가 유지됩니다.
- 대상 앱이 **관리자 권한**으로 실행 중이면 키가 안 들어갑니다 → `관리자 권한으로 재실행` 버튼 사용.

## 멀티모니터 / 주의

- 4대 등 멀티모니터에서 좌표는 **가상 데스크톱 기준**으로 처리됩니다(보조 모니터 음수 좌표 지원).
- 영역/좌표 지정은 **선택한 모니터 위에** 오버레이가 뜹니다.
- 방향키 등 **확장 키는 SendInput(스캔코드+확장 플래그)** 로 보내, 일반 `pyautogui.press` 가 무시되는 앱에서도 동작합니다.
- pyautogui **FAILSAFE** 활성: 마우스를 화면 좌상단으로 옮기면 자동화가 중단됩니다(안전장치).
- 트리거 핫키를 페이지 넘김 키와 **같게 두지 마세요**(자기 트리거 위험). 기본 트리거는 `F8` 입니다.
